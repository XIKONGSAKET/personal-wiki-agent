# Exploration Profile

For thinking a topic through when there's **no build intended yet** — understanding a space, weighing options, forming a view. The output is clarity, not a project folder.

## When to use

- "Help me think about / understand X"
- Weighing an open question with no immediate thing to build
- Mapping a space before deciding whether to act at all

If a build or a concrete decision is the goal, switch to the matching preset (`personal-project`, `startup`, `personal-decisions`, …).

## Flow shape

- **Phase 1 — Understanding:** `full` — what are you actually trying to figure out?
- **Phase 2 — Context:** `full` — this is the heart of an exploration (map the landscape, the evidence, the angles).
- **Phase 3 — Generation:** `full` — framings / options / hypotheses, not products.
- **Phase 4 — Deep Dive:** `light` — go deep only on the angle the user cares about.
- **Phase 5 — Critique:** `light` — pressure-test the *thinking* (assumptions, missing angles, weak evidence), not a build. A short, pointed verdict — not the full venture premortem.
- **Phase 6 — Plan:** `skip` by default — there's nothing to build. If the user wants next steps, offer them inline, but don't dispatch the isolated planner.
- **Scaffold:** off — there's no build to hand off.

If, mid-exploration, the user decides to build or to make a decision, tell them you're switching preset and **wait** for them to confirm (§2) before changing the flow.

## Phase customizations

### Phase 1 — Understanding
- What question are you really trying to answer?
- What would a satisfying answer look like — a decision, a model, a map?
- What do you already believe, and how strongly?

### Phase 2 — Context (the core)
- Map the landscape: what's known, what's contested, what's unknown.
- Tier 3 search: current facts, data, expert views — cite sources.
- Surface the strongest cases on more than one side.

### Phase 3 — Generation
- Generate framings, hypotheses, or options — distinct ways to see the topic, not products to build.

### Phase 4 — Deep Dive
- Go deep only where the user points.

### Phase 5 — Critique (light)
- Stress-test the reasoning: unexamined assumptions, missing perspectives, weak evidence, confirmation bias. Short and sharp.

## Frameworks the coordinator may invoke

- First-principles / 5 Whys
- Steelmanning multiple positions
- Second-order effects
- What Needs to Be True
- Expected value (for weighing options)

## Anti-patterns specific to exploration

- Pushing toward a build or a plan the user didn't ask for.
- Running the full venture-grade premortem on something that isn't a venture.
- Forcing a go/no-go verdict when the user wants understanding, not a decision.
