# PDF分块阅读规则 v3.0

> 创建于 2026-04-15  
> 更新于 2026-04-15  
> 核心改进：从"固定页数分块"改为"按小标题结构分块"

---

## 核心原则：**逻辑完整性优先**

```
旧方式（已废弃）：固定10页分块 ❌
    问题：会截断章节，混合不同主题
    
新方式（现行）：按小标题结构分块 ✅
    优势：每个chunk都是自洽的逻辑单元
```

---

## 第一步：结构探查（必读）

### 1.1 读取策略

```
操作：先读取PDF的前10-15页（或目录页）
目的：识别文档的章节结构、标题层级
输出：建立"阅读地图"，列出所有小标题及其页码范围
```

### 1.2 目录识别

**期刊论文**（通常有明确结构）：
- Abstract / 摘要
- Keywords / 关键词
- 1. Introduction / 引言
- 2. Literature Review / 文献综述
- 3. Methodology / 方法
- 4. Results / 结果
- 5. Discussion / 讨论
- 6. Conclusion / 结论

**博士论文**（中国知网格式）：
- 前置部分（封面、声明、摘要）- 罗马数字页码
- 目录（通常在PDF第8-15页）
- 第1章 绪论
- 第2章 理论基础
- 第3章 实证分析
- ...

**书籍/专著**：
- 需要查找目录页（通常在PDF前20页）
- 记录各章节起始页码

## 第二步：分块策略（核心改进）

### 2.1 分块单位

**最小单位**：小标题（二级或三级标题）

```
示例结构：
第2章 理论基础
├── 2.1 概念界定        ← Chunk 1
├── 2.2 理论框架        ← Chunk 2
│   ├── 2.2.1 理论A     ← Chunk 3（如内容长）
│   └── 2.2.2 理论B     ← Chunk 4（如内容长）
└── 2.3 研究假设        ← Chunk 5
```

### 2.2 动态调整规则

| 情况 | 处理策略 |
|------|---------|
| 一个小标题 **< 3页** | 与下一个相邻小标题**合并**为一个chunk |
| 一个小标题 **3-10页** | 作为一个**独立chunk** |
| 一个小标题 **> 10页** | **细分**为多个子chunk（按逻辑段落） |
| 跨越章节边界 | **禁止** - 必须在章节结束处切割 |

### 2.3 禁止切割点

以下位置**严禁**切割：
- 一个论证的中间（如：论据1 → [切割] 论据2 → 结论）
- 一个案例/实验的描述中间
- 一段引用的中间
- 图表及其解释文字之间
- 公式推导的中间

## 第三步：分块执行流程

### 3.1 标准流程

```python
# 伪代码表示
def process_pdf(pdf_path):
    # 步骤1: 探查结构
    toc_pages = extract_toc(pdf_path)  # 提取目录
    structure = parse_headings(toc_pages)  # 解析标题层级
    
    # structure = [
    #   {"title": "2.1 概念界定", "level": 2, "start_page": 15, "end_page": 18},
    #   {"title": "2.2 理论框架", "level": 2, "start_page": 19, "end_page": 28},
    #   ...
    # ]
    
    # 步骤2: 智能分块
    chunks = []
    current_chunk = None
    
    for heading in structure:
        page_count = heading["end_page"] - heading["start_page"] + 1
        
        # 规则：小标题<3页，尝试合并
        if page_count < 3:
            if current_chunk and can_merge(current_chunk, heading):
                current_chunk = merge(current_chunk, heading)
            else:
                if current_chunk:
                    chunks.append(current_chunk)
                current_chunk = create_chunk(heading)
        # 规则：小标题3-10页，独立成块
        elif page_count <= 10:
            if current_chunk:
                chunks.append(current_chunk)
                current_chunk = None
            chunks.append(create_chunk(heading))
        # 规则：小标题>10页，细分
        else:
            if current_chunk:
                chunks.append(current_chunk)
                current_chunk = None
            sub_chunks = subdivide(heading)  # 按逻辑段落细分
            chunks.extend(sub_chunks)
    
    # 步骤3: 逐个读取并处理
    for chunk in chunks:
        content = read_pdf(pdf_path, chunk["start_page"], chunk["end_page"])
        summary = digest_chunk(content, chunk["title"])
        save_summary(summary)
        
    return chunks
```

## 第四步：特殊情况处理

### 4.1 无法识别目录的情况

如果PDF没有清晰的目录或标题层级：

1. **尝试读取前20页**：寻找隐含的标题（字体加粗、字号变化）
2. **人工判断**：如果我无法识别结构，向用户汇报：
   ```
   "该PDF结构不清晰，建议：
    - 选项A：告诉我主要章节标题和页码，我按此分块
    - 选项B：我按段落大致分块，可能精度较低"
   ```

### 4.2 博士论文的特殊处理

中国知网博士论文特点：
- 目录在PDF第8-15页（罗马数字页码区）
- 正文章节从"第1章"开始（阿拉伯数字页码）
- 每章可能包含：引言、理论、方法、案例、小结

**处理策略**：
```
读取目录 → 识别每章的小节 → 按二级/三级标题分块
例如：
- 第3章 实证研究（页码1-50）
  - 3.1 研究区域（页码1-5）→ Chunk
  - 3.2 数据来源（页码6-8）→ Chunk（与3.1合并？判断）
  - 3.3 分析方法（页码9-15）→ Chunk
  - 3.4 结果分析（页码16-40）→ 细分多个chunks
  - 3.5 本章小结（页码41-50）→ Chunk
```

### 4.3 图表密集区域

如果某个chunk内图表密集（>3个连续页面以图表为主）：
- 标注：`⚠️ 该区域图表密集，建议人工查看`
- 记录图表编号和标题
- 不跳过，但降低文字摘要的权重

## 第五步：质量检查

### 5.1 每个Chunk的自检问题

在生成摘要前，问自己：

1. **逻辑完整性**：这个chunk是否包含了一个完整的论证/描述？
2. **独立性**：脱离上下文，这个chunk是否能被理解？
3. **无截断**：没有截断案例、公式、引用吧？
4. **边界清晰**：与前后chunk的边界是否明确？

### 5.2 自动核对与执行

**无需用户确认**，自动生成并执行分块方案。


## 附录：与旧规则(v2.0)的主要区别

| 维度 | 旧规则(v2.0) | 新规则(v3.0) |
|------|-------------|-------------|
| **分块单位** | 固定10页 | 按小标题（动态） |
| **切割依据** | 页码数字 | 逻辑结构（标题层级） |
| **最小chunk** | 10页（固定） | 3页（可合并） |
| **最大chunk** | 10页（强制切割） | 10页（建议细分） |
| **处理顺序** | 先读→再分块 | 先探查结构→再分块→再读取 |
| **用户确认** | 可选 | ❌ **已取消，改为自动核对** |
| **自检机制** | 无 | ✅ **自动生成+自检后直接执行** |
