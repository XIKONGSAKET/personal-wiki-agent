# Skill 命令：update-X.X.X.X

从用户提供的DOCX文件中提取指定章节（如2.1.3），并更新到对应的论文记忆文件中。

**命令格式**：
```
/update-X.X.X.X "DOCX完整路径"
```

**示例**：
```
/update-2.1.3 "C:\obsidian\LLM WIKI test01\inspool-wiki-zh\raw\unprocessed\260423 江门市老旧小区更新规划研究.docx"
```

**执行前必须首先阅读 `Academic-wiki-agents-rules.md` 和 `inspool-wiki-zh/meta/DOCX增量处理使用指南.md` 获取详细规则。**

## 执行步骤

### Phase 1: 参数解析与验证

1. **提取章节编号**：从命令名中解析章节号（如 `update-2.1.3` → `2.1.3`）
2. **获取DOCX路径**：从用户输入中提取DOCX文件的完整路径
3. **验证文件存在**：确认DOCX文件存在于指定路径
4. **确定记忆文件位置**：
   - 从DOCX文件名提取论文名称
   - 构建记忆文件路径：`inspool-wiki-zh/wiki/meta/论文记忆-{论文名}.md`

### Phase 2: 提取章节内容

**使用 docx_ingest.py 脚本提取指定章节**：

```bash
python "inspool-wiki-zh/scripts/docx_ingest.py" "DOCX路径" --section "章节号" --output "temp/chapter_X_X_X.md"
```

**示例**：
```bash
python "inspool-wiki-zh/scripts/docx_ingest.py" "C:\obsidian\LLM WIKI test01\inspool-wiki-zh\raw\unprocessed\260423 江门市老旧小区更新规划研究.docx" --section "2.1.3" --output "inspool-wiki-zh/temp/chapter_2_1_3.md"
```

**提取失败处理**：
- 如果脚本返回错误或输出为空，报告具体错误信息
- 不继续后续步骤，等待用户确认

### Phase 3: 读取并结构化内容

1. **读取提取的章节文件**（`temp/chapter_X_X_X.md`）
2. **内容结构化**：
   - 识别章节标题层级
   - 提取核心论点与关键内容
   - 整理关键引用和参考文献
   - 保留原始段落结构

**结构化模板**：
```markdown
#### X.X.X 章节标题
**状态**：🟢 已完成

**核心内容**：

[章节引言/概述]

**（1）小节标题**
[内容要点]

**（2）小节标题**
[内容要点]

**关键引用**：
- 作者[编号]：引用说明
- ...
```

### Phase 4: 更新记忆文件

1. **读取现有记忆文件**（`wiki/meta/论文记忆-XXX.md`）
2. **定位目标章节**：找到对应章节号的位置（如 `#### X.X.X`）
3. **替换内容**：
   - 将占位符内容（🔴 空/占位符）替换为结构化内容
   - 更新章节状态为 🟢 已完成
4. **更新待办事项**：
   - 找到 `- [ ] 填充X.X.X...` 项
   - 改为 `- [x] 填充X.X.X...`
5. **更新时间戳**：
   - 更新 frontmatter 中的 `updated: YYYY-MM-DD`
   - 更新最后更新说明：`*最后更新：YYYY-MM-DD - 更新第X.X.X章（标题）完整内容*`

### Phase 5: 验证与报告

**内容验证**：
- [ ] 章节内容是否成功提取？
- [ ] 记忆文件是否正确更新？
- [ ] 章节状态是否从 🔴 变为 🟢？
- [ ] 待办事项是否已勾选？
- [ ] 时间戳是否已更新？

**输出报告**：

```
## 章节更新报告

**源文件**：`DOCX完整路径`
**目标章节**：X.X.X
**记忆文件**：`wiki/meta/论文记忆-XXX.md`

**更新内容摘要**：
| 小节 | 核心内容 |
|------|---------|
| （1）XXX | ... |
| （2）XXX | ... |

**状态变更**：🔴 空/占位符 → 🟢 已完成
**待办事项**：已勾选 X.X.X 的填充任务
**最后更新**：YYYY-MM-DD
```

## 约束与禁止事项

### 绝对禁止

1. **严禁编造内容**
   - 必须使用docx_ingest.py实际提取DOCX内容
   - 不得根据章节标题推测内容
   - 提取失败必须明确报错，不能蒙混过关

2. **严禁跳过验证**
   - 必须确认DOCX文件存在才能执行
   - 必须确认记忆文件存在才能更新
   - 提取失败不得继续更新步骤

3. **路径规范**
   - DOCX路径必须使用用户提供的路径
   - 不因文件重命名/版本变化而自动推断路径
   - 记忆文件路径基于DOCX文件名推导

### 错误处理

| 场景 | 处理方式 |
|------|---------|
| DOCX文件不存在 | 报告错误，停止执行 |
| docx_ingest.py提取失败 | 报告错误信息，停止执行 |
| 记忆文件不存在 | 提示用户先创建记忆文件（使用"记住当前状态"） |
| 章节内容为空 | 报告提取结果为空，停止执行 |
| 目标章节不存在于记忆文件 | 报告章节不匹配，询问是否新建 |

## 章节编号格式支持

| 命令格式 | 章节层级 | 示例 |
|---------|---------|------|
| `/update-X` | 一级章节（第X章） | `/update-2` → 第2章 |
| `/update-X.X` | 二级章节 | `/update-2.1` → 2.1节 |
| `/update-X.X.X` | 三级章节 | `/update-2.1.3` → 2.1.3小节 |

## 故障排查

### DOCX提取失败常见原因

| 现象 | 原因 | 建议 |
|------|------|------|
| 提取内容为空 | 章节标题格式不匹配 | 检查DOCX中的标题层级样式 |
| 提取到错误内容 | 章节编号重复或嵌套 | 确认DOCX结构是否规范 |
| Python脚本错误 | python-docx未安装 | 运行 `pip install python-docx` |

### 记忆文件更新失败

| 现象 | 原因 | 建议 |
|------|------|------|
| 找不到目标章节 | 记忆文件结构与预期不符 | 检查记忆文件中的章节编号 |
| 替换后格式错乱 | 原始占位符格式不一致 | 手动调整或报告问题 |
